Sales Email Drafts

Video walkthroughs for creating advertising email drafts across our publications. Watch each to learn the process, then use the templates to get started.

Creating First Emails for a New Slate

21 min

Walks through the end-to-end workflow for generating first-email drafts for a new HAVEN campaign slate using the New Drafts Creation Sheet, Slack triggers, Lindy AI automation, and Google Apps Script.

Trigger: Create drafts Path: New Themes:
  1. Review and select themes/events from the Themes document (e.g., PGA Championship, Suncoast Boat Show) for the campaign.
  2. Prepare the New Drafts Creation Sheet with the correct keyword ("New Themes:") so the Lindy agent follows the right automation path.
  3. Copy theme data for each market (New England, Philadelphia, Tampa, Palm Beach, Arizona Southwest, Las Vegas, Hawaii) into the spreadsheet.
  4. Post the prepared rows into the Draft Creation Slack channel, triggering the Lindy AI workflow to auto-generate first-email drafts per market.
  5. Collect generated drafts in the First Emails Creation Doc, apply formatting via Apps Script, and verify trigger words, seasonal language, events, and feeder markets.
  6. Split drafts into separate documents per market using another Apps Script.
  7. Create updated data notes for the slate (e.g., "spring market" messaging).
  8. Review and lightly edit drafts — event wording, feeder markets, etc.
  9. Send drafts for internal review (Sophie → Usama → Yujia channel for testing).

PA Creation + RE Creation

18 min

Explains creating PA (Past Advertiser) email drafts using the PA Creation Doc and RE (follow-up) email drafts using the RE Creation Doc for a HAVEN campaign slate after the first email drafts are completed.

Trigger: Create drafts Path: RE:
  1. Create PA drafts by copying the same market drafts (e.g., New England, Philadelphia, Tampa, Palm Beach, Arizona, Las Vegas, Hawaii) into the PA Creation Doc.
  2. Run the PA formatting script, which adjusts the email wording to reflect that the recipient has advertised previously, adding familiarity and relationship-based language.
  3. Update specific lines manually, such as replacing the closing question with wording like: "Would you like me to share rate details or pull up your last layout?"
  4. Copy the finalized PA drafts into each market's document for use in upcoming campaigns.
  5. Generate RE (follow-up) drafts by modifying the Slack path to "RE:", which tells the Lindy AI agent to create follow-up emails based on the existing first email themes.
  6. Paste the market links into the Draft Creation Slack channel, allowing Lindy to analyze the first emails and automatically generate RE drafts for each theme.
  7. Organize and format the RE drafts using Apps Script and adjust pricing or promotional offers (e.g., BOGO deals or different price points depending on the market).
  8. Review deals and spam-filter considerations (removing bold/italic formatting).
  9. Save RE drafts in the RE Creation Doc, then later paste them into each market's draft document before the follow-up emails are scheduled to send.

Creating Supplementary Drafts

17 min

Demonstrates creating supplementary email drafts using the Supplementary Draft Creation Doc and the Supplementary Draft Creation Sheet — similar to generating RE drafts but requiring additional manual adjustments for edition variables, market names, and seasonal language.

Trigger: Create drafts Path: Supplementary:
  1. Identify the slate markets (e.g., Miami, Fort Lauderdale, Upstate, Minnesota, Canada, Vancouver, California).
  2. Trigger the Lindy AI automation by posting draft links into the Draft Creation Slack channel using the path "Supplementary:" to generate supplementary drafts.
  3. Open and organize the generated drafts by market in the Supplementary Draft Creation Doc.
  4. Adjust edition variables and market names (e.g., Homes + Lifestyles, specific city editions) to ensure the correct distribution language.
  5. Copy the drafts into a master document, labeling sections by market.
  6. Run a formatting script to separate and organize the drafts automatically.
  7. Manually update certain sections using content from the previous campaign slate to ensure consistency with existing templates.
  8. Verify trigger keywords (e.g., "spring market") and update seasonal language where needed.
  9. Update special drafts such as Under $300K first email and RE drafts.
  10. Remove bold/italic formatting to standardize the email style.
  11. Copy finalized supplementary drafts into the Data Notes documents for each market so the sales team can use them.

Stats Report Generation

Step-by-step process for generating advertiser performance reports. The workflow branches depending on the report type and whether the advertiser already has an article in place.

1. Receive the Request & Identify the Advertiser

New stats requests come in via the Monday.com stats request board (and a dedicated Slack channel called stats requests). A sales rep fills out a form and the request appears on the board.

  1. Find the contact in Salesforce — Search for the advertiser in Salesforce. Note: names may differ between the request and Salesforce (e.g., "Kenny" vs. "Kenneth"). Check Gmail for name variations if needed.
  2. Open their most recent deal record — Look at the deal history and identify the most recent issue code (e.g., FLP-12125). Note the publish date and issue details.
  3. Copy the Deal ID — Extract the Deal ID from the Salesforce URL for use in the stats form later.
  4. Find the article — Check the Pagination tab on the deal record for an online article link. You can also search for the advertiser's article on the Haven website back end, or check the Google Drive layout folder linked in the deal record.

2. Determine the Report Path

Based on what you find, you'll follow one of four paths. The path depends on whether the advertiser has an article and whether their individual ad exists in Facebook Ads Manager.

  1. Path A — Individual Article Report — The advertiser has an article on the website and their specific ad exists in Facebook Ads Manager. You generate a report scoped to their individual article. When exporting from Ads Manager, keep the ad set name filter in place, add age demographic, and export the CSV.
  2. Path B — Market / Issue Report — The advertiser has an article, but their individual ad does not appear in Facebook Ads Manager. You generate a full market/issue report instead. When exporting from Ads Manager, remove the ad set name (so all ad sets in the campaign are included), add age demographic, and export the CSV. The report will show overall market performance rather than individual article stats.
  3. Path C — No Article Exists — No article was ever created for the advertiser (e.g., they didn't fill out property details). You need to create the article first on the website, then either create a Facebook ad campaign yourself at the same daily budget as other ads or send the article URL to Vine to have it added. Notify the sales rep that stats will be available in 2–3 days once the ad accumulates data, then come back and pull the stats.
  4. Path D — Non-Real-Estate / Vertical Ad — The advertiser is a vertical (e.g., construction, furniture, paving) rather than a real estate listing. These don't fit the standard report template. Ask Serphie if there's a banner ad or article for the advertiser. If stats can be pulled, generate them normally but format the stats as bullet points in a text message for the sales rep instead of a full report link. A dedicated vertical report template is being developed.

3. Export Facebook Ads Data

Open Meta Ads Manager under the Haven team account. Find the campaign matching the market and issue month (e.g., "FLP January").

  1. Navigate to the correct campaign — Campaigns are named by market and month (e.g., "FLP January"). Search or filter to find the right one.
  2. Search for the advertiser's article — Look for their article title within the campaign's ad sets. Make sure the date range is wide enough to show everything.
  3. If the individual ad exists (Path A) — Keep the ad set name filter in place. Add the Age demographic breakdown. Export as CSV.
  4. If the individual ad does NOT exist (Path B) — Select the full market campaign. Remove the ad set name field. Add the Age demographic breakdown. Export as CSV. This gives you the overall market stats instead.
  5. Create a custom report — Click "Create New Report," configure the columns (remove ad set name for market reports, keep it for individual), add age demographic, and export.

4. Export Issuu Magazine Stats

Go to Issuu and find the publication matching the issue and publish date from Salesforce.

  1. Find the issue — Navigate to Publications on Issuu. Search for the publication by market and date (e.g., "Haven Central North Florida" published 1/23/2026).
  2. Open the statistics — Go to the Statistics tab for that issue. Set the date range to All Time to maximize the impressions captured.
  3. Download the PDF — Download the stats as a PDF. This provides magazine impressions, reading metrics, and distribution data.
  4. Copy the Issue URL — You'll also need the publication URL and the cover image. For the cover image, inspect the page source and search for og:image to find the image URL.

5. Submit to the Stats Tool

Open the stats reporting form and upload all the data you've collected.

  1. Upload the Meta/Facebook CSV — The CSV you exported from Ads Manager.
  2. Upload the Issuu stats PDF — The PDF you downloaded from Issuu.
  3. Enter the publish date — The date the issue was published online (from Salesforce or Issuu), formatted as MM/DD/YYYY.
  4. Enter the Issue publication URL — The Issuu URL for the digital magazine.
  5. Enter the magazine cover image URL — Found by inspecting the Issuu page source and searching for og:image. Can also be found on the Haven website. If left blank, check whether it auto-populates; if not, you can add it manually to the spreadsheet afterward.
  6. Enter the Salesforce Deal ID — Extracted from the deal record URL in Salesforce.
  7. Select the report type — Choose Individual Article (Path A) or Market / Issue Report (Path B) depending on whether the advertiser's specific ad was found in Ads Manager.
  8. Enter the advertiser name — Must match exactly as it appears on the article.
  9. Paste the article source code — Open the article page, view page source (not the console), copy the full HTML, and paste it into the form. This extracts the agent image, property details, and article metadata.
  10. Submit the form — Click process. The form posts data to a Google Sheet and generates the report. A new row is created (or an existing row is updated if the Deal ID already exists).

6. Verify & Deliver the Report

Once the form processes, it generates a report link. Check that everything looks right, then send it to the sales rep.

  1. Hard refresh the report page — After the form processes, open the generated report URL and do a hard refresh (Shift+Cmd+R on Mac) to ensure the latest data displays.
  2. Check the report contents — Verify that all links work, images load correctly (especially the magazine cover), the landing page is correct, and the stats look reasonable. The report pulls total impressions from the Issuu PDF, clicks and demographics from the Facebook CSV, and article/agent details from the source code.
  3. Fix the cover image if missing — If the magazine cover image didn't auto-populate, add it manually to the spreadsheet in the designated column (AF), then hard refresh the report.
  4. Send the report link to the sales rep — Find the requesting rep on the Monday.com board. Send them the plain-text report URL so they can copy/paste it into an email for the advertiser. Include the issue name (e.g., "January–February 2026 Report").
  5. For Path D (vertical ads) — If a full report doesn't apply, format the stats as bullet points and send them as a text message to the sales rep instead of a report link.
