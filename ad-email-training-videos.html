Creating First Emails for a New Slate

21 min

Walks through the end-to-end workflow for generating first-email drafts for a new HAVEN campaign slate using the New Drafts Creation Sheet, Slack triggers, Lindy AI automation, and Google Apps Script. Required trigger: "Create drafts"; Path keyword: "New Themes:".

  1. Review and select themes/events from the Themes document (e.g., PGA Championship, Suncoast Boat Show) for the campaign.
  2. Prepare the New Drafts Creation Sheet with the correct keyword ("New Themes:") so the Lindy agent follows the right automation path.
  3. Copy theme data for each market (New England, Philadelphia, Tampa, Palm Beach, Arizona Southwest, Las Vegas, Hawaii) into the spreadsheet.
  4. Post the prepared rows into the Draft Creation Slack channel, triggering the Lindy AI workflow to auto-generate first-email drafts per market.
  5. Collect generated drafts in the First Emails Creation Doc, apply formatting via Apps Script, and verify trigger words, seasonal language, events, and feeder markets.
  6. Split drafts into separate documents per market using another Apps Script.
  7. Create updated data notes for the slate (e.g., "spring market" messaging).
  8. Review and lightly edit drafts — event wording, feeder markets, etc.
  9. Send drafts for internal review (Sophie → Usama → Yujia channel for testing).

PA Creation + RE Creation

18 min

Explains creating PA (Past Advertiser) email drafts using the PA Creation Doc and RE (follow-up) email drafts using the RE Creation Doc for a HAVEN campaign slate after the first email drafts are completed. Required trigger: "Create drafts"; Path keyword: "RE:".

  1. Create PA drafts by copying the same market drafts (e.g., New England, Philadelphia, Tampa, Palm Beach, Arizona, Las Vegas, Hawaii) into the PA Creation Doc.
  2. Run the PA formatting script, which adjusts the email wording to reflect that the recipient has advertised previously, adding familiarity and relationship-based language.
  3. Update specific lines manually, such as replacing the closing question with wording like: "Would you like me to share rate details or pull up your last layout?"
  4. Copy the finalized PA drafts into each market's document for use in upcoming campaigns.
  5. Generate RE (follow-up) drafts by modifying the Slack path to "RE:", which tells the Lindy AI agent to create follow-up emails based on the existing first email themes.
  6. Paste the market links into the Draft Creation Slack channel, allowing Lindy to analyze the first emails and automatically generate RE drafts for each theme.
  7. Organize and format the RE drafts using Apps Script and adjust pricing or promotional offers (e.g., BOGO deals or different price points depending on the market).
  8. Review deals and spam-filter considerations (removing bold/italic formatting).
  9. Save RE drafts in the RE Creation Doc, then later paste them into each market's draft document before the follow-up emails are scheduled to send.

Creating Supplementary Drafts

17 min

Demonstrates creating supplementary email drafts using the Supplementary Draft Creation Doc and the Supplementary Draft Creation Sheet — similar to generating RE drafts but requiring additional manual adjustments for edition variables, market names, and seasonal language. Required trigger: "Create drafts"; Path keyword: "Supplementary:".

  1. Identify the slate markets (e.g., Miami, Fort Lauderdale, Upstate, Minnesota, Canada, Vancouver, California).
  2. Trigger the Lindy AI automation by posting draft links into the Draft Creation Slack channel using the path "Supplementary:" to generate supplementary drafts.
  3. Open and organize the generated drafts by market in the Supplementary Draft Creation Doc.
  4. Adjust edition variables and market names (e.g., Homes + Lifestyles, specific city editions) to ensure the correct distribution language.
  5. Copy the drafts into a master document, labeling sections by market.
  6. Run a formatting script to separate and organize the drafts automatically.
  7. Manually update certain sections using content from the previous campaign slate to ensure consistency with existing templates.
  8. Verify trigger keywords (e.g., "spring market") and update seasonal language where needed.
  9. Update special drafts such as Under $300K first email and RE drafts.
  10. Remove bold/italic formatting to standardize the email style.
  11. Copy finalized supplementary drafts into the Data Notes documents for each market so the sales team can use them.

Using the Stats Tool

10 min

Explains how to use HAVEN's stats reporting tool to generate performance reports for advertisers. The process involves locating the advertiser's listing in Salesforce, verifying the article and property details, exporting performance data from Facebook Ads Manager and Issuu, and uploading that data into a reporting form that automatically generates an analytics report. The final report link is then shared with the requesting sales rep so they can send it to the advertiser.

  1. Identify the Advertiser Request — Stats report requests usually come from sales reps via Slack or message. Find the advertiser in Salesforce to determine which issue and listing should be used for the report.
  2. Locate the Listing in Salesforce — Open the advertiser's deal record in Salesforce. Check the issues they ran in, whether they have a subscription, and which issue is appropriate to generate stats from. Choose a recent issue with enough time for distribution so meaningful data exists.
  3. Verify the Article — In the Salesforce deal record, go to the Pagination tab and copy the article link. Open the article page and check that the listing appears correctly, property details look accurate, and the landing page works properly. Optionally verify the property on Zillow to confirm price accuracy. If updates are needed, request edits from Hammad Khan.
  4. Export Facebook Ads Data — Open Meta Ads Manager. Navigate to the campaign for article promotions, find the specific article campaign, select the maximum date range, add the Age demographic metric, and export the report as CSV. This provides impressions, clicks, and demographic data.
  5. Export Issuu Magazine Stats — Locate the relevant issue in Issuu (User: havenlifestyles / Password: TimBrown81!). Confirm the advertiser appears in the issue, open the issue analytics, select the maximum date range, and download the statistics. This provides magazine impressions and reading metrics.
  6. Upload Data to the Stats Tool Form — Open the stats reporting form. Upload the Issuu stats CSV and Facebook Ads CSV. Enter additional details: publication date, Salesforce deal ID, and advertiser name (must match article exactly).
  7. Extract Article Data — Open the article page in incognito mode. View page source, copy the source code, and paste it into the stats tool form. This allows the tool to extract property details, article metadata, and additional analytics data.
  8. Generate the Report — Submit the form. The scripts will process the uploaded data, combine digital and magazine stats, and generate a performance report. Review the output to confirm the metrics appear reasonable.
  9. Send the Report — Open the generated Analytics Report URL. Send the report link to the requesting sales rep (e.g., "HAVEN January–February 2026 Report"). The sales rep can then forward this report directly to the advertiser.
  10. Optional Data Verification — If needed, open the Stats Data Sheet. This sheet contains all report data fields and can be used to verify metrics, manually update data, or troubleshoot discrepancies.
